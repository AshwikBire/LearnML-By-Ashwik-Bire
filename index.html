ML Job Interview Preparation

Prepare for machine learning interviews with common questions and answers.

Technical Concepts

What is the bias-variance tradeoff?

The bias-variance tradeoff is a fundamental concept in machine learning that describes the tension between two sources of error:

  • Bias: Error from erroneous assumptions in the learning algorithm. High bias can cause underfitting.
  • Variance: Error from sensitivity to small fluctuations in the training set. High variance can cause overfitting.

The goal is to find a balance where both bias and variance are minimized to achieve optimal model performance.

Explain the difference between bagging and boosting.

Bagging (Bootstrap Aggregating):

  • Creates multiple models in parallel using bootstrap samples of the training data
  • Combines predictions through averaging (regression) or voting (classification)
  • Reduces variance without increasing bias
  • Example: Random Forest

Boosting:

  • Creates models sequentially where each new model focuses on the errors of previous ones
  • Combines weak learners into a strong learner
  • Reduces bias and can also reduce variance
  • Examples: AdaBoost, Gradient Boosting, XGBoost

Algorithm Questions

How does a random forest algorithm work?

Random Forest is an ensemble learning method that operates by constructing multiple decision trees during training and outputting the mode of the classes (classification) or mean prediction (regression) of the individual trees.

Key features:

  • Uses bagging (bootstrap aggregating) to create diverse trees
  • Introduces additional randomness when growing trees by selecting a random subset of features at each split
  • Reduces overfitting compared to individual decision trees
  • Provides feature importance estimates
What is the kernel trick in SVM?

The kernel trick allows Support Vector Machines to operate in a high-dimensional, implicit feature space without ever computing the coordinates of the data in that space, but rather by simply computing the inner products between the images of all pairs of data in the feature space.

This approach:

  • Makes SVMs capable of solving non-linear classification problems
  • Avoids the computational expense of explicitly mapping data to high dimensions
  • Uses kernel functions like polynomial, RBF (radial basis function), and sigmoid

Practical Scenarios

How would you approach a project with imbalanced classes?

When dealing with imbalanced classes, I would consider several approaches:

Data-level approaches:

  • Oversampling the minority class (e.g., using SMOTE)
  • Undersampling the majority class
  • Collecting more data for the minority class

Algorithm-level approaches:

  • Using algorithms that handle imbalance well (e.g., tree-based algorithms)
  • Adjusting class weights or decision thresholds
  • Using appropriate evaluation metrics (precision, recall, F1-score, AUC-ROC instead of accuracy)

Ensemble methods:

  • Using balanced bagging or boosting techniques
How do you handle missing values in a dataset?

The approach to handling missing values depends on the nature and pattern of the missingness:

For small amounts of missing data:

  • Remove instances with missing values (if the percentage is very small)
  • Remove features with high percentage of missing values

For larger amounts of missing data:

  • Mean/median/mode imputation for numerical data
  • Mode imputation for categorical data
  • More sophisticated methods like k-NN imputation or regression imputation
  • Algorithm-based imputation like MICE (Multiple Imputation by Chained Equations)

Advanced approaches:

  • Using algorithms that handle missing values internally (e.g., XGBoost)
  • Treating missingness as a feature itself
